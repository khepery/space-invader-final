1. Getting Started

Ultra Pro Space Invaders is a browser-based educational game that combines classic arcade action with quiz-based learning. Students destroy enemy ships while answering curriculum-aligned questions to earn power-ups, build combos, and compete on the leaderboard.

What You Need

  • A modern web browser (Chrome, Edge, Firefox, or Safari)
  • No internet required after initial load — the game runs entirely offline
  • A computer, tablet, or large touchscreen display (recommended for classroom use)

Opening the Game

  1. Open the game URL in your browser, or open the provided HTML file directly.
  2. You will see the Access Code screen.
  3. Enter the code included with your purchase and click LAUNCH GAME.
  4. You're in! The code is remembered by the browser for next time.

2. Access Code & Login

The game is protected by a simple access code to ensure only authorized users can play. Your access code was provided in your Teachers Pay Teachers purchase confirmation.

  • Default code: included in your purchase receipt.
  • The code is not case-sensitive — "galaxy2026" and "GALAXY2026" both work.
  • Once entered correctly, the browser remembers it — students won't need to re-enter it each time (unless they clear browser data).
Bookmark the game URL on all classroom devices so students can access it quickly.

3. Game Overview

Players pilot a spaceship at the bottom of the screen. Enemy ships descend from above, and asteroids drift across the battlefield. Players shoot enemies, dodge hazards, answer quiz questions, and collect power-ups.

Game Flow

  1. Intro Screen — "LAUNCH MISSION" to begin or view the leaderboard.
  2. Mission Configuration — Choose difficulty, load questions, select categories.
  3. Gameplay — Destroy enemies wave by wave. Questions appear regularly.
  4. Wave Boss — Each wave ends with a mini-boss encounter + quiz.
  5. Game Over — See your score, accuracy, grade, and save to the leaderboard.

Mouse

  • Move: Move the mouse left/right over the game canvas — the ship follows smoothly.
  • Shoot: Hold down the left mouse button to fire continuously.

Touchscreen

  • Move: Touch and drag on the game canvas — the ship follows your finger.
  • Shoot: The ship fires automatically while your finger is on the screen.
  • On-screen ◀ ▶ arrow buttons also appear on touch devices for alternative control.

Answering Questions

  • When a quiz appears, click/tap one of the four answer buttons (A, B, C, or D).
  • A timer counts down — answer before it expires!
  • The game pauses while the question is displayed.

5. Setting Up Questions

The game comes with 20 default questions across Math, Science, Geography, Literature, Art, and History. You can replace or supplement them with your own curriculum-specific questions.

Option A — Import a Text File

  1. On the Mission Configuration screen, click "Download Template" to get the format.
  2. Open the downloaded question_template.txt in any text editor.
  3. Add your questions following the format below.
  4. Click "Load My Questions" and select your file.
Question: What is the capital of Canada?
A: Toronto
B: Ottawa
C: Vancouver
D: Montreal
Correct: B
Category: Geography

Question: What is 15 × 12?
A: 170
B: 175
C: 180
D: 185
Correct: C
Category: Math
Separate each question with a blank line. Lines starting with # are treated as comments and ignored.

Option B — Add Manually

  1. Click "Add Manually" on the Mission Configuration screen.
  2. Type the question, four answer options, select the correct answer, and (optionally) a category.
  3. Click SAVE.

Option C — Use Default Questions

Click "Use Default Questions" to reset back to the built-in set of 20 questions.

Viewing & Deleting Questions

Click "View Questions" to see all loaded questions. Each question has a delete button if you want to remove individual entries.

You need at least 3 questions to start the game. The game cycles through questions and reshuffles them once all have been used.

9. Leaderboard

After each game, students can save their score with a name (up to 20 characters). The leaderboard is stored in the browser's local storage — it persists between sessions on the same device.

  • Access the leaderboard from the intro screen or the game over screen.
  • Use the 🗑 Clear button to reset the leaderboard (e.g., for a new class).
Have students save their names after each game to track progress over time. The leaderboard is per-device, so each classroom computer maintains its own rankings.

10. Touchscreen & Split-Screen Setup (2 Players)

This game is designed to work beautifully on large touchscreen displays. The recommended competitive setup uses your operating system's split-screen mode to run two game instances side by side, allowing two students to compete simultaneously.

Step-by-Step: Split-Screen on a Large Touchscreen

  1. Open two browser windows — open the game URL in one window, then open a second window with the same URL. (Tip: use Ctrl+N or Cmd+N for a new window.)
  2. Snap windows side by side:
    • Windows: Drag one window to the left edge of the screen until it snaps to fill the left half. Then select the other window for the right half. (Or use Win+← and Win+→.)
    • macOS: Hover over the green full-screen button → choose "Tile Window to Left/Right".
    • Chromebook: Drag the window to the left/right edge, or use Alt+[ and Alt+].
  3. Enter the access code in each window and launch the game.
  4. Configure each game independently — each student can pick their own difficulty, load their own questions, and compete!
  5. Each student plays on their half of the touchscreen — touch input is per-window, so each student's finger controls only their side.
A 55" interactive whiteboard running Windows 11 in split-screen mode: Student A plays on the left half, Student B plays on the right half. Both touch their own side of the screen to move their ship and answer questions. At the end, they compare scores on their respective game-over screens.
For the best competitive experience, have both students start their games at the same time with the same difficulty and the same question set. Whoever gets the higher score wins!

Full-Screen Mode (Single Player)

For single-player use, click the ⛶ fullscreen button (bottom-left corner) or press F to maximize the game. Press Esc to exit fullscreen.

11. Using Two Mice (Dual-Mouse Mode)

If you don't have a touchscreen, you can still run the two-player split-screen setup using two USB mice. Here's how it works:

How It Works

By default, most operating systems merge all connected mice into one cursor. To give each mouse its own independent cursor (so each student controls their own game window), you need special software:

Windows — Recommended Tools

ToolNotes
TeamPlayer Free multi-mouse tool for Windows. Creates a separate cursor for each connected mouse. Each cursor can click and interact with different windows independently.
Pluralinput Commercial multi-user input software. Supports multiple mice and keyboards. Good for classroom setups.
Windows MultiPoint Server Microsoft's built-in multi-station solution (available in some education editions).
MouseMux Lightweight tool that gives each mouse an independent cursor in Windows.

Setup Steps (with multi-cursor software)

  1. Connect two USB mice to the computer.
  2. Install and start your chosen multi-cursor tool (e.g., TeamPlayer).
  3. You should now see two independent cursors on screen.
  4. Open two browser windows and snap them side by side (split-screen).
  5. Each student uses their mouse to control their game window — clicking, moving the ship, and answering questions independently.
Two students sit at one computer. Each has their own USB mouse. With TeamPlayer running, Student A's mouse has a blue cursor on the left game window, and Student B's mouse has a red cursor on the right game window. They race through waves and compare scores at the end!

Alternative: One Mouse + One Keyboard

If two-mouse software isn't available, one student can use the mouse (move + click to shoot) while the other uses the keyboard (arrow keys + spacebar) — but this only works in a single game instance, not split-screen.

12. Frequently Asked Questions

Can I use my own curriculum questions?

Yes! Import a .txt file with your questions or add them manually. See Section 5.

Does the game require internet?

No. After the initial load, the game runs entirely offline. The single-file release version works with no server at all — just open the HTML file.

Can I use this on iPads or tablets?

Yes. The game is responsive and works on any device with a modern browser. Touchscreen controls activate automatically.

How many questions should I add?

Minimum 3 to start. We recommend 15–30 questions for a good session. The game cycles through and reshuffles once all are used.

Can students cheat by looking at the source code for answers?

The questions are loaded dynamically and can be custom each time. Since you control the question file, you can change it between classes.

What happens if a student gets a question wrong?

They lose a small amount of health (never enough to die from it alone) and the correct answer is displayed. The focus is on learning, not punishment.

Can I share this with other teachers?

No. The license is for the purchasing teacher's classroom only. Other teachers should purchase their own copy through Teachers Pay Teachers.

13. Troubleshooting

ProblemSolution
Game shows "Loading…" and doesn't start Make sure all image and audio files are in the same folder as the game. If using the single-file release, try a different browser.
No sound Click anywhere on the page first (browsers require user interaction before playing audio). Check the 🔊 sound button is enabled.
Access code doesn't work The code is not case-sensitive. Make sure there are no extra spaces. Check your purchase receipt for the correct code.
Questions won't import Verify your .txt file follows the exact format (see Section 5). Each question must have a "Question:", four options (A:–D:), and a "Correct:" line.
Game runs slowly Close other browser tabs. The game performs best in Chrome or Edge. Make sure hardware acceleration is enabled in browser settings.
Touch controls not responding in split-screen Make sure each window is focused by tapping on it first. Some OS touch implementations need a first tap to activate a window.
Leaderboard data lost Leaderboard data is stored in the browser's local storage. Clearing browser data, using incognito mode, or switching browsers will reset it.

14. Changing the Access Code

You can change the access code at any time by editing a single file:

  1. Open the file config.js in any text editor (Notepad, VS Code, etc.).
  2. Find the line: code: "GALAXY2026"
  3. Replace GALAXY2026 with your new code (keep the quotes).
  4. Save the file and redeploy.
// config.js — Change the access code here
window.SITE_CONFIG = {
  code: "YOUR_NEW_CODE_HERE"
};
After changing the code, tell your students the new code. Students who previously logged in with the old code will need to enter the new one.
Change the code periodically (e.g., every semester) to prevent former students from sharing it. Any word or phrase works — keep it simple and memorable for your class.
